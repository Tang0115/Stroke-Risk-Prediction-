## Setup

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd stroke-risk-prediction
```

### 2. Create a virtual environment

```bash
# Windows (PowerShell)
python -m venv .venv
.venv\Scripts\activate

# macOS / Linux
python -m venv .venv
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the dataset

Pick **one** of the two options below.

#### Option A ‚Äî Manual download

Download the CSV from Kaggle:  
https://www.kaggle.com/datasets/fedesoriano/stroke-prediction-dataset/data

Place the file here:
```
data/raw/healthcare-dataset-stroke-data.csv
```

#### Option B ‚Äî Kaggle API (auto-download)

Set up credentials **once per machine**, then the Phase 1 notebook downloads the
CSV for you via `src/data.py` (`ensure_dataset()`):

1. Create a free Kaggle account.
2. Go to **Account ‚Üí Settings ‚Üí API ‚Üí Create New API Token**. This downloads `kaggle.json`.
3. Place `kaggle.json` here:
   - **Windows:** `C:\Users\<you>\.kaggle\kaggle.json`
   - **macOS / Linux:** `~/.kaggle/kaggle.json` (then `chmod 600 ~/.kaggle/kaggle.json`)
4. Install the downloader (already in `requirements.txt`): `pip install kagglehub`

Run the first cells of `01_eda_cleaning.ipynb` ‚Äî if the CSV is missing it pulls it
automatically. If it already exists, nothing is downloaded.

> The raw data file is excluded from the repo by `.gitignore`, so each teammate
> supplies it locally via Option A or B.

## Running the Notebooks

Run the notebooks **in order**. Each phase depends on the output of the previous one.

| Step | Notebook | Output |
|------|----------|--------|
| 1 | `notebooks/phase1_eda/01_eda_cleaning.ipynb` | `data/processed/stroke_cleaned.csv`, figures in `results/figures/` |
| 2 | `notebooks/phase2_baseline/02_baseline_logistic.ipynb` | `results/metrics/phase2_baseline_metrics.csv` |
| 3 | `notebooks/phase3_advanced_models/03_advanced_models.ipynb` | `results/metrics/phase3_model_comparison.csv`, feature importance figures |
| 4 | `notebooks/phase4_final_analysis/04_final_analysis.ipynb` | Final comparison chart, conclusions |

### How to launch Jupyter

```bash
jupyter notebook
```

Then navigate to the `notebooks/` folder in the browser UI and open notebooks in order.

### VS Code

Open the `.ipynb` files directly in VS Code. Select the `.venv` Python kernel when prompted.

## Troubleshooting

**ImportError for `imblearn`:**  
```bash
pip install imbalanced-learn
```

**Notebook can't find the dataset:**  
Make sure `data/raw/healthcare-dataset-stroke-data.csv` exists. All paths in the notebooks are relative to the notebook file's location.

**Kernel not found in VS Code:**  
Open the Command Palette ‚Üí *Python: Select Interpreter* ‚Üí choose the `.venv` environment.
